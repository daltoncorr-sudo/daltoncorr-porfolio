# Content Migration Brief
## What goes where in the rebuild

---

## PURPOSE

This document maps every known project to its treatment in the new site. It's the checklist that ensures nothing gets lost in the migration from Squarespace → static HTML.

Once the site scrape spreadsheet is complete, cross-reference it against this document. Every row in the spreadsheet should have a home in the new site. Every project below should have corresponding content in the spreadsheet.

---

## MIGRATION STATUS KEY

- **MIGRATE** — Content exists on current site. Move it to the new site.
- **CREATE** — New project page. Content doesn't exist yet. Needs writing + assets from D.
- **COMBINE** — Multiple current pages merge into one new page.
- **ARCHIVE** — Do not migrate. Too old, too thin, or superseded.
- **DUPLICATE** — Remove. Identical or near-identical to another page.

## CONTENT THAT MOVES BETWEEN PAGES

### Press links: About page → Press page + Project pages
All press currently on the about page needs to be:
1. Centralized on the new Press page
2. Also linked on the relevant project page's press section

### Bio: About page → New About page
Rewrite needed. Current bio says "18th, 19th, and 20th" — needs to say "18th through 21st" at minimum, or better yet, stop counting and say "art director for the Oscar-qualifying HollyShorts Film Festival."

### Embedded media: Music pages → Work project pages
YouTube/Vimeo/SoundCloud embeds from current music pages need to be captured in the scrape and re-embedded on the new unified Work pages.

## IMAGE MIGRATION NOTES

1. All images currently on Squarespace CDN (`images.squarespace-cdn.com`) need to be downloaded at highest available resolution
2. Re-optimize for the new site (WebP with JPEG fallback, 3 sizes per image)
3. Rename files with consistent convention: `[project-slug]-[descriptor]-[number].jpg` (e.g., `hollyshorts-20-poster-01.jpg`, `hollyshorts-20-merch-hoodie.jpg`)
4. Store in `/assets/images/[project-slug]/` directories
5. The scrape spreadsheet's Images sheet is the master list — every image URL in that sheet must be accounted for (downloaded, renamed, placed, or intentionally excluded)
